# Contributing to EduTrack

Thank you for your interest in contributing to EduTrack! We appreciate your support.

---

## 🛠 How to Contribute

1. **Fork** the repository
2. **Clone** your fork:
   ```bash
   git clone https://github.com/yourusername/EduTrack.git
   ```
3. **Create a branch** for your feature or fix:
   ```bash
   git checkout -b feature/my-awesome-feature
   ```
4. **Make your changes**
5. **Write tests** for your changes (if applicable)
6. **Run tests** to verify:
   ```bash
   dotnet test
   ```
7. **Commit your changes**:
   ```bash
   git commit -m "Add my awesome feature"
   ```
8. **Push your branch**:
   ```bash
   git push origin feature/my-awesome-feature
   ```
9. **Open a Pull Request** and describe your changes

---

## 🧹 Code Style

- Follow the `.editorconfig` for C# style
- Use PascalCase for public members
- Use meaningful naming
- Prefer dependency injection over static usage
- Keep methods short and focused

---

## 📄 Pull Request Guidelines

- Link issues in PR if applicable
- Keep commits clean and descriptive
- Squash commits if needed
- Ensure all checks pass before requesting a review

---
